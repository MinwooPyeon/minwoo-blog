## 프로젝트 개요

물류 환경에서 **AMR(자율이동로봇) 자율주행 + 위험 감지 + 실시간 알림**을 End-to-End로 통합한 시스템입니다.  
임베디드(ROS2) — 시뮬레이터(Unity) — 백엔드(Spring Boot) — 안드로이드(FCM/RTSP)까지 하나의 파이프라인으로 연결했습니다.

| 항목 | 내용 |
|------|------|
| 개발 기간 | 2025.07.07 ~ 2025.08.20 |
| 팀원 | 6명 (임베디드·시뮬레이션·AI·백엔드·모바일) |
| 담당 역할 | **임베디드** — ROS2 제어 및 센서 융합, MQTT |
| 주요 성과 | 가상 센서 처리 117ms → 3.5ms, AI→서버→모바일 실시간 알림 파이프라인 |

## 주요 기능

### AMR 자율주행 & 통신

- ROS2 기반 SLAM · 경로 계획 · 모터 제어 노드
- MQTT 양방향 통신으로 AMR 상태/명령 실시간 교환

### 실시간 위험 감지 & 알림

- YOLO 기반 위험 상황 탐지(안전 장비 미착용, 물류 붕괴, 흡연, 쓰러짐)
- 탐지 → 서버 수신 → FCM 모바일 푸시 알림 파이프라인

### 디지털 트윈 시뮬레이터

- SLAM 맵 연계 Occupancy Grid 시각화
- Unity Burst + Job System으로 가상 센서 처리 **117ms → 3.5ms** 최적화
- 다중 AMR 충돌 회피 (리더 선점 · 경로 재탐색)

### 모바일 모니터링

- RTSP 기반 실시간 카메라 뷰 (ExoPlayer)
- FCM + Room + Flow로 알림 이력 동기화 / 재현성 확보

## 문제 해결

### 웹캠/RTSP 지연

실시간 스트림 처리 병목이 원인이었습니다.  
IPC + RTSP 혼합, 멀티프로세싱/큐 적용으로 해결했습니다.

### Unity 프레임 드랍

단일 스레드 CPU 병목 문제였습니다.  
Burst + Job System 병렬화로 **117ms → 3.5ms**로 개선했습니다.

### Jenkins in Docker CD 실패

컨테이너 내부에서 Docker에 접근할 수 없었습니다.  
Jenkins 이미지에 Docker 설치 + `/var/run/docker.sock` 바인딩으로 해결했습니다.

### 알림 이력 손실/불일치

서버–로컬 비동기 처리로 인한 문제였습니다.  
서버 저장 + Room 업서트 + Flow 구독으로 UI 자동 반영을 구현했습니다.
